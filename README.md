# Задание 1. Модуль детекции и отслеживания положения объекта в видеопотоке

## Описание проекта

Данный программный модуль предназначен для определения положения объекта в видеопотоке с веб-камеры в реальном времени. Модуль использует нейросеть YOLOv8 для детекции объектов и OpenCV для обработки видео и визуализации результатов.
Основные возможности:

   * Захват видеопотока с веб-камеры в реальном времени

   * Детекция объектов с использованием предобученной модели YOLOv8

   * Интерактивный выбор отслеживаемого объекта непосредственно в процессе работы

   * Определение и отображение параметров положения объекта:

       + Координаты центра объекта

       + Размеры ограничивающей рамки (bounding box)

       + Смещение объекта относительно центра кадра

   * Работа в реальном времени с низкой задержкой

   * Визуализация результатов детекции непосредственно на видеокадре

## Требования к системе

* Python 3.8 или выше

* Веб-камера (встроенная или внешняя)

* Операционная система: Windows/Linux/macOS

## Установка и настройка

1. Клонирование репозитория

```bash

git clone <url-репозитория>
cd <директория-проекта>
```

2. Установка зависимостей

Установите необходимые Python-пакеты:
```bash

pip install opencv-python ultralytics numpy

```

3. Загрузка модели YOLO

Модель YOLOv8n (yolov8n.pt) будет автоматически загружена при первом запуске программы. При необходимости вы можете использовать другие версии модели:

*    yolov8n.pt - нано-версия (самая быстрая)

*    yolov8s.pt - малая версия

*    yolov8m.pt - средняя версия

*    yolov8l.pt - большая версия (более точная, но медленная)

*    yolov8x.pt - экстра-большая версия

## Использование

Запуск программы

```bash

python opencv_example.py

```
Интерактивный режим работы

* Запуск программы: При запуске открывается окно с видеопотоком с веб-камеры.

* Выбор объекта для отслеживания:

    + Нажмите клавишу s для входа в режим выбора объекта

    + На экране отобразятся все обнаруженные объекты с номерами

    + Нажмите цифру (0, 1, 2...), соответствующую нужному объекту

    + Для выхода из режима выбора без выбора нажмите ESC

* Отслеживание объекта:

    + После выбора объекта программа начнет отслеживать его положение

    + На экране отображается:

        - Зеленый прямоугольник вокруг объекта

        - Красная точка - центр объекта

        - Синяя точка - центр кадра

        - Текстовая информация: Название класса и уверенность детекции, Смещение объекта относительно центра кадра (dx, dy), Координаты центра объекта, Размеры ограничивающей рамки.

* Смена отслеживаемого объекта:

    + В любой момент можно нажать s для выбора нового объекта

* Завершение работы:

    + Нажмите клавишу q для выхода из программы

# Задание 2. Симуляция движения робота в ROS2 с виртуальным лидаром

## Описание проекта

Симуляция реализована в среде Gazebo, urdf-описание робота подготовлено при помощи встроенного инструмента в САПР SolidWORKS для конвертации 3д-моделей в urdf. Приводы колёсной платформы реализованы при помощи ros2_control. Также реализован модуль локализации, позволяющий построить трёхмерную карту окружения и определить пространственное положение робота.

Используемые ROS2-пакеты:

   * LIO-SAM - модуль лидарно-инерцальной одометрии. Необходим для построения трёхмерной карты и определения пространственного положения робота.

   * velodyne_simulator - пакет, содержащий urdf-описание лидара velodyne и плагина для его симуляции.

   * wheeled_description - пакет, содержащий urdf-описание колёсного робота и launch-файл для запуска всей системы.

## Установка и настройка

1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd <директория-проекта>
```

2. Сборка изображения и контейнера

Для удобства сборки docker-контейнера и взаимодействия с ним реализованы bash-скрипты:

* build_img.sh - сборка изображения контейнера

* build_cont.sh - сборка самого контейнера

* new_window_cont.sh - открытие нового терминала в контейнере


## Использование

Для запуска симуляции соберите контейнер и инициализируйте рабочее пространство:

```bash
. install/setup.bash
```

Далее запустите launch-файл:

```bash
ros2 launch wheeled_description wheeled.launch.py

```

Для управления роботом откройте новое окно контейнера и выполните следующую команду:

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard /cmd_vel:=/wheeled_base_controller/cmd_vel_unstamped

```

После выхода из контейнера и завершения работы выполните очистку контейнера:
```bash
docker system prune
```
